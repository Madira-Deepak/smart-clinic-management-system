# Smart Clinic Management System User Stories

## Admin User Stories

### Admin Story 1

**User Story:**
As an Admin, I want to add doctors so that patients can book appointments.

**Priority:** High
**Story Points:** 5

**Acceptance Criteria:**

* Admin can enter doctor details.
* Doctor record is saved successfully.
* Doctor appears in the doctor list.

---

### Admin Story 2

**User Story:**
As an Admin, I want to update doctor information so that records remain accurate.

**Priority:** High
**Story Points:** 3

**Acceptance Criteria:**

* Admin can edit doctor details.
* Changes are saved successfully.
* Updated information is displayed correctly.

---

### Admin Story 3

**User Story:**
As an Admin, I want to remove inactive doctors so that the system remains current.

**Priority:** Medium
**Story Points:** 3

**Acceptance Criteria:**

* Admin can select a doctor for removal.
* Doctor record is deleted successfully.
* Removed doctor no longer appears in searches.

---

### Admin Story 4

**User Story:**
As an Admin, I want to view all appointments so that clinic operations can be monitored.

**Priority:** High
**Story Points:** 5

**Acceptance Criteria:**

* Admin can access appointment reports.
* All appointments are displayed.
* Appointment details include doctor and patient information.

---

### Admin Story 5

**User Story:**
As an Admin, I want to manage patient accounts so that records stay organized.

**Priority:** Medium
**Story Points:** 5

**Acceptance Criteria:**

* Admin can view patient accounts.
* Admin can update patient information.
* Changes are reflected immediately.

---

# Doctor User Stories

### Doctor Story 1

**User Story:**
As a Doctor, I want to view my appointments so that I can prepare for patient visits.

**Priority:** High
**Story Points:** 5

**Acceptance Criteria:**

* Doctor can access appointment list.
* Appointment details are visible.
* Appointments are sorted by date and time.

---

### Doctor Story 2

**User Story:**
As a Doctor, I want to update prescriptions so that patients receive proper treatment instructions.

**Priority:** High
**Story Points:** 5

**Acceptance Criteria:**

* Doctor can create a prescription.
* Doctor can edit prescription details.
* Prescription is linked to an appointment.

---

### Doctor Story 3

**User Story:**
As a Doctor, I want to manage my availability so that patients can book suitable appointment slots.

**Priority:** High
**Story Points:** 3

**Acceptance Criteria:**

* Doctor can add available slots.
* Doctor can remove unavailable slots.
* Updated schedule is visible to patients.

---

### Doctor Story 4

**User Story:**
As a Doctor, I want to access patient history so that I can provide informed treatment.

**Priority:** Medium
**Story Points:** 5

**Acceptance Criteria:**

* Doctor can view previous appointments.
* Doctor can view prescriptions.
* Patient history is displayed securely.

---

### Doctor Story 5

**User Story:**
As a Doctor, I want to receive appointment notifications so that I stay informed of schedule changes.

**Priority:** Medium
**Story Points:** 3

**Acceptance Criteria:**

* Notifications are generated for new appointments.
* Notifications are generated for cancellations.
* Doctor can view notifications.
